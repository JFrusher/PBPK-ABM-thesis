# Security Policy

## Supported Versions

This repository is research software and is maintained on a best-effort basis.

Security updates are prioritized for the default branch.

## Reporting a Vulnerability

Please do not open public issues for security-sensitive findings.

Instead:

1. Prepare a minimal report with impact, reproduction steps, and affected files.
2. Share the report privately with maintainers.
3. Allow reasonable time for validation and remediation before public disclosure.

## Disclosure Process

- Maintainers will acknowledge receipt as soon as practical.
- We will triage severity and affected scope.
- If confirmed, a patch and advisory timeline will be coordinated.

## Research Environment Notes

This project may execute external binaries and process large data outputs.
Use trusted datasets and run in controlled environments where possible.
